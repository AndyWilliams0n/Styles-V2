CSS Namespaces

This naming convention allows the css to ascertain exactly what kind of job a class might have, how and where it might be able to be reused, whether or not we can modify it, and more.

The types of namespace used here are:

  • 'c-' for Components
  • 'o-' for Objects
  • 'u-' for Utilities
  • 't-' for Themes
  • 'js-' for Javascript Classes and Tests
  • is-/has- for States

BEM Naming

BEM works by breaking all classes in a codebase down into one of three groups:

  • Block: The sole root of the component
  • Element: A component part of the Block
  • Modifier: A variant or extension of the Block

Blocks, Elements, and Modifiers: BEM. Absolutely every class in a project fits into one of these categories, which is why BEM is so great—it’s incredibly simple and straightforward.

The point of BEM is to give a lot more transparency and clarity in your markup. BEM tells developers how classes relate to each other, which is particularly useful in complex or deep pieces of DOM.

Hiding content

Content can be hidden using the class .u-hidden.

Classes are also available to show and hide content based on media query breakpoints.

These classes follow the patterns .u-hidden-*--up, .u-hidden-*--only and .u-hidden-*--down for devices ≥, == or ≤ the specified size respectively.
